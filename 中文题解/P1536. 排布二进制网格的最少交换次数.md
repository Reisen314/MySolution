1536. 排布二进制网格的最少交换次数

## 思路

看到这个最小交换次数，想到逆序数等于最小交换次数的结论，所以可以通过算逆序数得到答案，这里放一下这个概念的定义。

**逆序数**是指在一个排列中，前面的数大于后面的数的情况。换句话说，如果在一个排列中，一对数的前后位置与大小顺序相反，那么它们就构成一个逆序对。一个排列中所有逆序对的总数叫做这个排列的逆序数。

逆序数一定是最小交换次数的证明网上有很多，这里就不放了。

然后通过从上到下，每次选序号最小的合法行（这个步骤的合法性在下面证明），确定好解，再计算逆序数得到最小交换次数。

刚开始想了一会没思路，感觉拆解成几个问题可能有救，不过我不擅长巧妙的拼接子问题，结果最后写了几个辅助函数。

## 复杂度

时间复杂度: $$O(n^2)$$ 


空间复杂度: $$O(n)$$


## 证明
最小交换次序实际上等于这个排列的逆序数(e.g. 1342->1234 逆序数2，都由2提供)，假设我们在第i行时，可以选j1或j2交换(不妨设j1<j2)，假如最优解是交换j2，那么j1自然在第i行后的某处被用到，如果这样，我们可以交换j1和j2，这样解决了一个逆序对，逆序数-1，比假设的最优解更小，则假设不成立，所以最优解只能是选j1。由此，选最近的一定是最优策略。

当然这里用到这道题的性质，就是某一行可以放在第i行，也可以放在第i行下面的任意行，这个性质保证了我们交换j1和j2得到的解是合法的



```python

#从后往前数零的个数
def count_zero(nums):
    n=len(nums)
    ans=0
    for i in range(n-1,-1,-1):
        if not nums[i]:
            ans+=1
        else:
            return ans
    return ans
#计算逆序数，想不到好办法，暴力算了
def count_reverse(nums):
    n=len(nums)
    cnt=0
    for i in range(1,n):
        for j in range(i):
            if nums[j]>nums[i]:
                cnt+=1
    return cnt
class Solution:
    def minSwaps(self, grid: List[List[int]]) -> int:
        count=defaultdict(list)
        n=len(grid)
        for i in range(n):
            # 只有前k个非零，可以在k-1行及以下使用
            k=n-count_zero(grid[i])
            count[k].append(i)
        #选sortedlist保证优先选到行编号最小的
        temp=SortedList()
        ans_list=[]
        #全是0的行先放好
        temp.update(count[0])
        for j in range(n):
            #每次放入可选的行
            temp.update(count[j+1])
            #temp空了表示有选不出来的情况，退出
            if not temp:
                return -1
            ans_list.append(temp.pop(0))
        #最后计算逆序数表示最小交换次数
        return count_reverse(ans_list)
```
